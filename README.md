# 15-Puzzle (Пятнашки)

Консольная реализация классической головоломки "Пятнашки" для Windows с автоматическим решением.

![C++](https://img.shields.io/badge/C%2B%2B-14-blue.svg)
![Platform](https://img.shields.io/badge/Platform-Windows-lightgrey.svg)
![IDE](https://img.shields.io/badge/IDE-Visual%20Studio-5C2D91.svg)

## Описание

Игра представляет собой квадратное поле 4×4 с 15 пронумерованными костяшками и одной пустой ячейкой. Цель — упорядочить костяшки по возрастанию, перемещая пустую ячейку с помощью стрелок.

При запуске автоматически генерируется случайная **разрешимая** позиция. Доступен режим автоматической сборки (solver), который находит оптимальное решение за считанные секунды.

## Возможности

- 🎲 **Случайная разрешимая позиция** при каждом запуске
- ⌨️ **Управление стрелками** — перемещение пустой ячейки
- 🤖 **Автоматическое решение** — поиск оптимального пути (клавиша `h`)
- 📜 **История ходов** с прокруткой (PgUp/PgDn)
- 🏆 **Экран победы** с возможностью просмотра истории
- 🎨 **Цветной вывод** — зелёные плитки на правильных местах, серые auto-ходы

## Управление

| Клавиша | Действие |
|---------|----------|
| `← ↑ → ↓` | Переместить пустую ячейку |
| `h` | Запустить автоматическое решение |
| `PgUp` | История назад (старые ходы) |
| `PgDn` | История вперёд (новые ходы) |
| `n` | Новая игра |

## Архитектура

```
Game (Orchestrator)
 ├── Board      — состояние поля, перемещение, перемешивание
 ├── History    — хранение и отображение истории ходов
 ├── Solver     — поиск оптимального решения (IDA*)
 └── Renderer   — отрисовка в консоли
```

### Solver: IDA* + Linear Conflict

Для поиска решения используется алгоритм **IDA/*** (Iterative Deepening A*) с эвристикой **Манхэттенское расстояние + Линейный конфликт**.

| Алгоритм | Память | Скорость |
|----------|--------|----------|
| A* + Manhattan | O(миллионы состояний) | Минуты |
| **IDA* + Manhattan + Linear Conflict** | **O(длина решения)** | **0.1–3 секунды** |

**Линейный конфликт** — дополнение к Манхэттену: если две плитки в одной строке/столбце должны остаться в ней, но стоят в обратном порядке, добавляется +2 к эвристике. Это ускоряет поиск в 50–100 раз.

## Сборка

### Требования

- Windows 10/11
- Visual Studio 2022 (или новее)
- MSVC с поддержкой C++14

### Инструкция

1. Клонировать репозиторий:
```bash
git clone https://github.com/username/15-puzzle.git
```

2. Открыть `Fifteen-puzzle.sln` в Visual Studio

3. Собрать проект: `Build → Build Solution` (или `Ctrl+Shift+B`)

4. Запустить: `Debug → Start Without Debugging` (или `Ctrl+F5`)

## Технические детали

- **Константа поля:** `BOARD_SIZE = 4` (переименована из `SIZE` для избежания конфликта с Windows API)
- **Проверка разрешимости:** `(inversions + blankRowFromBottom) % 2 == 1`
- **Перемешивание:** случайная перестановка + коррекция (swap двух ненулевых плиток при необходимости)
- **Хранение состояния:** `array<array<int, 4>, 4>` — совместимо с хеш-таблицами
